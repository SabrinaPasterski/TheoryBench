About TheoryBench.

TheoryBench brings a paper draft, PDF preview, computations, references, and Theoria chat into one local-first project room for theoretical physics.

What it is

A paper workspace

Edit project files, compile locally, inspect the PDF, and keep review comments beside the source.

A computation surface

Run Python and Wolfram cells as part of the working draft, with outputs promoted only when you choose.

An advisor layer

Theoria can find references, flag likely mistakes, and ask for the next useful check using your model accounts.

A share path

The first sharing target is an inspectable bundle: selected source, assets, traces, and a compiled PDF.

Local vs web

Runs on your machine

  • Project files and local sidecars
  • TeX compile and PDF rendering
  • Python cells and local packages
  • Mathematica or Wolfram Engine license
  • Claude/Codex CLIs or keys you configure

Shared only by choice

  • Read-only export bundle first
  • Overleaf import/export or Git sync later
  • Scoped OAuth only for sync targets
  • Separate view, comment, edit, trace, and run permissions
  • No default upload of private project data

How it works

  1. Open a project folder

    Start with an existing paper folder or a sample project. TheoryBench reads the source, assets, bibliography files, notes, and sidecars from that folder.

  2. Check the toolchain

    The local bridge reports TeX, Python, Wolfram activation, PDF rendering, and Claude/Codex routes before you rely on them.

  3. Write and compute in context

    Use LaTeX as the source of truth, run cells when a derivation needs a check, and promote only the results that belong in the paper.

  4. Ask Theoria at the source

    Theoria should attach feedback to lines, equations, citations, cells, or trace entries, then push toward the next computation rather than replacing the work.

  5. Export deliberately

    Share a read-only bundle first. Overleaf sync and OAuth belong to scoped import/export flows, not to remote execution of your local notebook.

Beta boundary

The workbench can run locally today, but the login/share workflow is still being hardened: full Overleaf sync, collaborator accounts, remote/sandboxed computation, and the derivation map extractor should be treated as active beta work.
