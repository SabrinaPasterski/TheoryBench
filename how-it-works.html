How it Works

TheoryBench is one file. Run it, and a workbench opens in your browser — editor, PDF compile, Python and Wolfram cells, and the Theoria reviewer. Everything executes on your machine.

There is no TheoryBench server. Projects, traces, and tokens stay on your computer.

From download to derivation

1. Run one file

Double-click the download. The workbench opens in your browser. No install, no account, no key.

2. A welcome project opens

Compile it, run its cells, let Theoria review it. Your projects live under ~/TheoryBench/.

3. Bring your own paper

Create a blank project, or import a folder or .zip from the Projects page. Your .tex files stay ordinary.

4. Overleaf, if you want it

Paste the project's Overleaf Git URL in the share panel; pull and push while collaborators keep writing there. Optional, never required.

5. Everything runs locally

Your TeX install compiles, your Python and Mathematica run the cells, your Claude account reviews.

6. Nothing leaves your machine

No TheoryBench server exists. The only network traffic is what you opt into: INSPIRE lookups and your own model accounts.
