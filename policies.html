Policies and requirements

Private work stays local by default. Export, sync, and model routes should be explicit project choices.

Local-first policy

Project data

Source, figures, notebooks, outputs, and traces live in your local project folder by default. Overleaf/Git sync can version ordinary paper files; private sidecars are excluded unless you deliberately include them.

Model routing

Claude and Codex use the user's configured account paths. The UI should label when a review used a CLI, API key, or no model route.

Compute routing

Python and Wolfram execution happens through the local bridge unless a future project explicitly chooses a trusted remote or sandboxed route.

Sharing

Overleaf remains the live collaborator workspace when you connect it. TheoryBench adds local compute, selected traces, compiled artifacts, and read-only bundles when requested.

Software and license requirements

TeX

  • Requires a local TeX install such as TeX Live or MacTeX.
  • Local package versions can differ from Overleaf.
  • Engine selection and BibTeX/Biber support should be explicit.

Wolfram

  • Requires Mathematica, Wolfram Engine, or another configured kernel route.
  • A local license or activation is the user's responsibility.
  • Wolfram Cloud support is a possible route, not the default assumption.

Python

  • Uses the local Python environment available to the bridge.
  • Project packages should be reported clearly when missing.
  • Cell state and provenance should stay document-scoped.

Accounts

  • Overleaf Git or Overleaf GitHub sync is the primary paper collaboration route.
  • GitHub Desktop can manage the local checkout when GitHub sync is used.
  • Claude, Codex/OpenAI, and Wolfram account connections are optional tool routes.
  • OAuth should be scoped to sync or sharing targets, not broad project upload.
  • No account should imply permission to upload or run arbitrary collaborator code.
